# Better Comments Convention

En caso de que un comentario sea estrictamente necesario para el contexto del desarrollo, utilizar exclusivamente las siguientes etiquetas:

- `TODO`: Para tareas pendientes o futuras implementaciones.
- `!`: Para alertas, advertencias críticas o código sensible.
- `?`: Para preguntas, dudas técnicas o lógica por definir.
- `*`: Para enfatizar información importante o puntos clave.

*Nota: Se debe respetar la jerarquía de colores configurada en el editor para estas etiquetas.*
